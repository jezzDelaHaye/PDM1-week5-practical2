Exercise 1.3

Your sketch:

Instructions

Create a sketch with a 400px by 400px canvas.

Virtual environments often contain obstacles that the player cannot move through. Your task is to create a simple environment with a moveable player object and an obstacle. The player object can be moved with the arrow keys but it can't pass through the obstacle. See the video below. The green square is the player and the black rectangle is the obstacle.

Create two global objects, one called player and one called obstacle.

Set the player properties as shown in the table below:

Key Value
x 50
y 50
size 50

Set the obstacle properties as shown in the table below:

Key Value
x 150
y 100
width 100
height 200

Draw a square to represent the player object and a rectangle to represent the obstacle. Use the default drawing mode for rectangles.

Make it possible to move the player using the usual w-a-s-d keys. The player should move smoothly and continuously if a w-a-s-d key is held down and stop moving if the key is released. It should not be possible for the player to overlap or pass through the obstacle.

When you're finished, check your work using check-exercise1-3.html.

Expected output:

To make the player move smoothly and continuously, use the keyIsPressed system variable rather than an event function.

To prevent the player from moving over the obstacle, you will need to add logic that checks if moving in the selected direction would make the player square overlap with the obstacle on any side. You have seen many similar examples of this type of logic--look at examples that check if a shape has collided with an edge of the canvas or if a point (such as the mouse) is in or out of bounds of some shape.

The logic is much easier if you write logic to work out if the player can move in the intended direction than if you write logic to work out if the player can't move. For example, on the y axis:

  • If the new position of the player is completely above the obstacle, it can move
  • If the new position of the player is completely below the obstacle, it can move
